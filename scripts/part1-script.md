# Part 1 脚本：Codex CLI で Web ページを作って GitHub Pages に公開しよう

> **収録メモ**
> - 目標尺：10〜15 分
> - 初心者向け。専門用語は初出時に必ず噛み砕いて説明する
> - `【画面】` は画面切り替え・キャプチャの目安
> - `【テロップ】` はテキスト表示で補足する情報
> - `（間）` は少し呼吸を置くポイント

---

## OP / フック（〜40 秒）

【画面】完成した Web ページがブラウザーで表示されている状態

---

「今日はこのページを、ゼロから作って、インターネット上に公開するところまで一気にやります。」

「しかも、コードはほとんど自分では書きません。AI にお願いするだけです。」

（間）

「使うのは Codex CLI（コーデックス・シーエルアイ）という AI ツールです。日本語で『こういうページを作って』とお願いすると、HTML ファイル――つまり Web ページの中身を自動で作ってくれます。」

「プログラミングの経験がなくても大丈夫です。一緒に手を動かしながら進めていきましょう。」

---

## Scene 1：今日の全体像（〜1 分）

【画面】フロー図 or ステップ一覧スライド
【テロップ】① リポジトリ作成 → ② 開発環境を起動 → ③ Codex で HTML 作成 → ④ 確認 → ⑤ GitHub に送る → ⑥ 公開

---

「今日やることの全体像を先にお見せします。」

「ステップは大きく分けて6つです。」

「まず、GitHub――ギットハブと読みます――というサービス上に、ファイルの置き場所を作ります。これを**リポジトリ**と呼びます。リポジトリは『プロジェクト専用のフォルダー』だと思ってもらえれば大丈夫です。」

「次に、ブラウザーの中で動くプログラミング用の作業スペースを開きます。これが **Codespaces** です。自分のパソコンに何かをインストールする必要はありません。」

「作業スペースが開いたら、AI エージェントの **Codex CLI** をセットアップして、Web ページを作ってもらいます。」

「できたページをブラウザーで確認して、GitHub に送って、公開設定をオンにする。これで世界中からアクセスできる自分のページが完成します。」

（間）

「それでは始めましょう。」

---

## Scene 2：GitHub アカウントの準備（〜1 分 30 秒）

【画面】github.com トップページ
【テロップ】GitHub：コードをインターネット上で保存・共有できるサービス

---

「まず GitHub のアカウントが必要です。すでにお持ちの方はそのままログインしてください。」

「まだの方は、github.com を開いて、右上の『Sign up』からアカウントを作成します。メールアドレス、ユーザーネーム、パスワードを入力して、画面の案内に従って進めるだけです。」

（間）

「GitHub というのは、プログラムのファイルをインターネット上に保存して、他の人と共有できるサービスです。世界中の開発者が使っていて、今回使う『Web ページをタダで公開できる機能』――GitHub Pages も、この GitHub の中にあります。」

---

## Scene 3：リポジトリを作る（〜2 分）

【画面】GitHub にログインした状態 → 右上「＋」→ New repository

---

「ログインできたら、ファイルの置き場所、つまりリポジトリを作ります。」

「右上の『＋』マークをクリックして、『New repository』を選びます。」

【画面】New repository の入力フォーム
【テロップ】リポジトリ ＝ プロジェクト専用のフォルダー。ファイルの変更履歴も自動で記録される

「Repository name のところに名前を入力します。何でもいいですが、今回は `my-site` にしましょう。」

（間）

「下にスクロールすると『Add a README file』というチェックボックスがあります。ここにチェックを入れてください。README というのは、プロジェクトの説明書きみたいなファイルです。チェックを入れておくと、最初から1つファイルが入った状態でリポジトリが作られるので、この後の操作がスムーズになります。」

「準備ができたら『Create repository』をクリック。」

【画面】作成されたリポジトリのトップページ（README.md が見えている状態）

「こんな画面になれば OK です。`README.md` というファイルが1つ入ったリポジトリができました。」

---

## Scene 4：Codespaces を起動する（〜2 分）

【画面】リポジトリのトップページ

---

「次に、プログラミングの作業スペースを開きます。ここで使うのが **GitHub Codespaces** です。」

【テロップ】Codespaces ＝ ブラウザーの中で動くプログラミング用の作業環境。インストール不要

「Codespaces は、クラウド上――つまりインターネットの向こう側にあるコンピューター上で動く開発環境です。見た目は VS Code というエディターそのもので、自分の PC に何もインストールしなくても、ブラウザーだけでコードを書いたり、コマンドを実行したりできます。」

（間）

「起動しましょう。リポジトリのページで、緑色の『Code』ボタンをクリックして、上部の『Codespaces』タブを選びます。そして『Create codespace on main』をクリックします。」

【画面】Codespaces 起動中の画面

「起動には 1〜3 分ほどかかります。画面はそのまま開いておいてください。」

【画面】Codespaces が開いた状態（VS Code 風の画面）

「開きました。画面の構成を簡単に説明すると――左がファイルの一覧、真ん中がファイルの中身を編集するエリア、そして下にあるのが**ターミナル**です。」

「ターミナルというのは、文字でコンピューターに命令を出す場所です。普段はマウスでクリックして操作しますが、ターミナルでは文字のコマンドを打ち込んで操作します。今日はこのターミナルをよく使います。」

---

## Scene 5：Codex CLI をインストールして起動する（〜2 分 30 秒）

【画面】Codespaces のターミナルにフォーカス

---

「それでは主役の登場です。Codex CLI をインストールします。」

「**Codex CLI** は、OpenAI が作った AI エージェントです。エージェントというのは、ただ質問に答えるだけでなく、ファイルを作ったり、コマンドを実行したり、実際の作業をこなしてくれる AI のことです。」

【テロップ】Codex CLI ＝ ターミナルで動く AI エージェント。自然言語で指示するとコードを書いてくれる

「ターミナルに以下のコマンドをコピー＆ペーストして、Enter を押してください。」

```
npm i -g @openai/codex
```

「`npm` というのは、JavaScript 系のツールをインストールするための仕組みです。このコマンドで Codex CLI がマシンにインストールされます。」

（間）

「インストールが完了したら、次のコマンドで Codex を起動します。」

```
codex
```

【画面】Codex のロゴが回転するアニメーション

「この回転するロゴが出たら起動成功です。」

（間）

「ここで認証が必要になります。ChatGPT の有料プランをお持ちの方は、そのアカウントでサインインできます。手順は概要欄にリンクを貼っておきます。API キーを使う方は、表示された入力欄にキーを貼り付けてください。」

【画面】認証成功後の画面

「認証が通ると、このような画面になります。」

（間）

### モデルの選択

「もうひとつだけ準備があります。Codex で使う AI のモデルを選びます。」

「Codex の画面で `/model` と入力してください。」

【画面】モデル選択画面
【テロップ】モデル ＝ AI の「頭脳」。タスクに合わせて使い分けるとコストと速度を最適化できる

「いくつかモデルが表示されます。今回は Web ページを作るというシンプルな作業なので、速くてコスパのいい `gpt-5.1-codex-mini` を選びます。キーボードの上下キーで移動して Enter です。」

「次に Reasoning Level を聞かれます。Reasoning というのは、AI がどれくらい深く考えてから答えを出すかの度合いです。複雑なバグ修正やアルゴリズムの設計では上げたほうがいいですが、今回は `default` で十分です。」

---

## Scene 6：Codex に HTML を作ってもらう（〜1 分 30 秒）

【画面】Codex CLI の入力画面

---

「準備が整いました。いよいよ Codex に Web ページを作ってもらいます。」

「以下の文章をそのまま Codex に入力して Enter を押してください。」

```
このリポジトリは、GitHub Pagesで公開する自分のWebサイトです。
index.htmlを新規作成し、シンプルなHTMLを用意してください。
```

【画面】Codex がファイルを生成していく様子

「Codex が動き始めました。`index.html` というファイルを自動で作ってくれています。」

「**HTML**――エイチティーエムエルと読みます――は、Web ページの骨組みを定義するための言語です。ブラウザーはこの HTML を読み取って、画面にページを表示します。`index.html` という名前は『トップページ』を意味する慣例的なファイル名です。」

【画面】左のファイルエクスプローラーに index.html が追加された状態

「左のファイル一覧に `index.html` が増えていれば成功です。クリックすると中身も確認できます。」

---

## Scene 7：ブラウザーで確認する（〜1 分 30 秒）

【画面】ターミナル右上の「＋」ボタン

---

「作ったページをブラウザーで見てみましょう。」

「ここで注意点がひとつ。Codex CLI が動いているターミナルはそのままにして、新しいターミナルを開きます。右上の『＋』ボタンをクリックしてください。`bash` と書かれたタブが追加されます。」

【テロップ】Codex 用のターミナルと、コマンド実行用のターミナルを分けて使う

「この新しいターミナルで、以下のコマンドを実行します。」

```
python3 -m http.server 5500
```

「これは Python という言語に付属している簡易 Web サーバーを起動するコマンドです。今いるフォルダーの中身を、ブラウザーから見られるようにしてくれます。」

【画面】「ブラウザーで開く」のポップアップが表示される

「ポップアップが出たら『ブラウザーで開く』をクリックします。」

【画面】ブラウザーに HTML ページが表示された状態

「表示されました。まだシンプルですが、ちゃんと動いています。確認できたら、サーバーを起動したターミナルで `Ctrl + C` を押して止めておきましょう。」

---

## Scene 8：GitHub に変更を送る（〜2 分 30 秒）

【画面】Codex CLI のタブに戻る

---

「ページが確認できたので、次はこのファイルを GitHub に送ります。この『送る』操作を **push（プッシュ）** と呼びます。」

「その前に、Git の基本的な流れをざっくり説明します。」

【テロップ】Git の3ステップ：① add（記録の準備）→ ② commit（記録する）→ ③ push（送る）

「Git では、ファイルの変更を記録・送信するのに3つのステップがあります。」

「まず **add**。これは『このファイルの変更を記録に含めますよ』という準備です。」

「次に **commit（コミット）**。これが『変更を正式に記録する』操作です。セーブポイントを作るようなイメージです。」

「最後に **push**。記録した内容をインターネット上の GitHub に送信します。」

（間）

「この3つを手動でやることもできますが、Codex に丸ごとお願いできます。」

「Codex に以下を入力してください。」

```
変更を確認してpushしてください
```

【画面】Codex が git add / commit / push を自動で実行していく様子

「Codex が自動で add、commit、push をやってくれます。途中でネットワークアクセスの許可を求められたら『Yes, proceed』を選んでください。」

【画面】push 成功後、GitHub のリポジトリページに index.html が追加されている

「GitHub のリポジトリページをリロードすると、`index.html` が追加されているのが確認できます。自分の PC からインターネット上の GitHub にファイルが届きました。」

---

## Scene 9：GitHub Pages で公開する（〜2 分）

【画面】リポジトリページ上部の「Settings」タブ

---

「いよいよ最後のステップ、Web ページの公開です。ここで **GitHub Pages** を使います。」

【テロップ】GitHub Pages ＝ GitHub にあるファイルを、そのまま Web サイトとして無料で公開できる機能

「GitHub Pages は、リポジトリに入っている HTML ファイルなどを、そのままインターネット上の Web サイトとして公開してくれる機能です。サーバーを自分で用意する必要はありません。」

（間）

「設定しましょう。リポジトリページの上部にある『Settings』をクリックします。」

【画面】左メニューから「Pages」を選択

「左側のメニューから『Pages』を選びます。」

【画面】Branch を main に変更 → Save

「『Branch』のところがデフォルトでは『None』になっています。これを `main` に変更して、『Save』をクリックします。」

「`main` というのは、このリポジトリのメインのブランチ――作業ラインのことです。ここでは『main ブランチにあるファイルを公開してね』と GitHub に伝えています。」

（間）

「設定を保存したら、反映まで 1〜3 分ほど待ちます。ページをリロードすると……」

【画面】「Your site is live at https://...」の表示

「『Your site is live at』の後に URL が表示されます。これが自分のページの公開アドレスです。クリックしてみましょう。」

【画面】公開されたページがブラウザーで表示される

「表示されました！ インターネット上に、自分の Web ページが公開されています。この URL を友達に送れば、誰でもアクセスできます。」

## 収録ノート

| シーン | 注意点 |
|-------|--------|
| Scene 4 | Codespaces の起動待ち（1〜3 分）はカット。起動後の画面にジャンプカットする |
| Scene 5 | 認証方法は視聴者によって異なる。詳細は概要欄のリンクへ誘導し、動画内では深追いしない |
| Scene 7 | ポップアップが出ない場合は VS Code の「PORTS」タブから手動で開ける旨をテロップ補足 |
| Scene 7 | ポート 5500 が使えない場合 → `5501` に変えれば OK、とテロップで補足 |
| Scene 8 | Codex が push の許可を求めるダイアログは確実にカメラに収める。初心者がここで止まりやすい |
| Scene 9 | Pages の反映待ち時間はカット。URL が表示された状態から再開する |
| Scene 9 | 404 が出る場合 → index.html がルート直下にあるか確認、とテロップ補足 |
